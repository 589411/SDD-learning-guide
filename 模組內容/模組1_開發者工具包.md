# 模組 1：開發者工具包 — 打造你的 AI 開發工場

## 🎯 核心目標
克服對專業開發工具的恐懼，建立一個能執行 SDD 流程的標準化環境。掌握版本控制、IDE、測試框架等現代開發必備工具。

## 📖 理論基礎

### 為什麼需要專業開發工具？

| 手動開發 | 工具化開發 |
|---------|----------|
| ⏰ 重複性工作耗時 | ⚡ 自動化節省時間 |
| 🐛 容易出錯 | ✅ 減少人為失誤 |
| 👤 依賴個人記憶 | 📝 系統化管理 |
| 😰 難以協作 | 🤝 標準化流程 |
| 🔙 無法回溯 | ⏮️ 版本追蹤 |

```
開發者
  ├── VS Code (編輯器)
  ├── Git (版本控制)
  ├── npm (套件管理)
  ├── Jest (測試框架)

```

### 💻 步驟 1：安裝 VS Code

#### 快速安裝
1. 前往 https://code.visualstudio.com/
2. 下載並安裝適合你作業系統的版本
3. 開啟 VS Code

#### 必裝擴充功能
在 VS Code 中按 `Cmd/Ctrl + Shift + X` 開啟擴充功能面板：

```

✅ Prettier - 程式碼格式化
✅ ESLint - 程式碼檢查
✅ GitLens - Git 增強功能
✅ Live Server - 即時預覽網頁（前端開發）
```

#### 基本設定
按 `Cmd/Ctrl + ,` 開啟設定，或直接編輯 `settings.json`：

```json
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.tabSize": 2,
  "files.autoSave": "afterDelay",
  "editor.wordWrap": "on"
}
```

#### 常用快捷鍵

| 快捷鍵 | 功能 | 使用情境 |
|-------|------|---------|
| `Cmd/Ctrl + P` | 快速開檔 | 快速找到並開啟檔案 |

### ⏰ 步驟 2：安裝 Git

#### 安裝 Git

**macOS:**
```bash
brew install git
```

**Windows:**  
下載並安裝：https://git-scm.com/download/win

**Linux (Ubuntu):**
```bash
sudo apt-get install git
```

#### 初始設定

```bash
# 設定你的名字和 Email（會出現在 commit 記錄中）
git config --global user.name "你的名字"
git config --global user.email "your.email@example.com"

# 設定預設分支名稱為 main
git config --global init.defaultBranch main

# 驗證設定
git config --list
```

#### Git 基本流程實戰

**場景：建立線上教師課程預約系統**

```bash
# 1. 建立專案資料夾並初始化 Git
mkdir course-booking-system
cd course-booking-system
git init

# 2. 建立規格資料夾
mkdir specs
echo "# 課程預約系統規格" > specs/README.md

# 3. 查看狀態
git status
# 會看到未追蹤的檔案（紅色）

# 4. 加入變更到暫存區
git add .

# 5. 提交版本
git commit -m "feat: 初始化專案並新增規格文件"

# 6. 查看歷史記錄
git log --oneline
```

#### Git 分支管理

**為什麼需要分支？**
- 🔀 獨立開發新功能，不影響主線
- 🤝 多人協作時避免衝突
- 🔙 失敗時可以輕鬆放棄

**實戰範例：**
```bash
# 查看目前分支
git branch

# 建立並切換到新分支「功能：教師時段設定」
git checkout -b feature/teacher-schedule

# 在新分支上開發
echo "## 教師時段設定規格" > specs/teacher-schedule.spec.md
git add .
git commit -m "feat: 新增教師時段設定規格"

# 切回主分支
git checkout main

# 合併功能分支
git merge feature/teacher-schedule

# 刪除已合併的分支（可選）
git branch -d feature/teacher-schedule
```

#### 連結 GitHub

```bash
# 1. 在 GitHub 建立新 Repository
# 網址：https://github.com/new

# 2. 連結遠端儲存庫
git remote add origin https://github.com/你的帳號/course-booking-system.git

# 3. 推送到 GitHub
git push -u origin main

# 4. 之後只需要
git push
```

#### Git 常用命令速查

| 命令 | 說明 | 使用時機 |
|------|------|---------|
| `git status` | 查看當前狀態 | 隨時檢查有哪些變更 |
| `git add .` | 加入所有變更 | 準備提交前 |
| `git commit -m "msg"` | 提交版本 | 完成一個階段性工作 |
| `git log --oneline` | 查看歷史 | 回顧開發記錄 |
| `git diff` | 查看變更內容 | 確認改了什麼 |
| `git checkout <branch>` | 切換分支 | 切換工作內容 |
| `git pull` | 拉取遠端更新 | 同步最新版本 |
| `git push` | 推送到遠端 | 備份或分享程式碼 |

#### Commit 訊息規範

**格式：**
```
<type>: <subject>

<body>（可選）
```

**常用 type：**
- `feat`: 新功能
- `fix`: 修復 bug
- `docs`: 文件更新
- `refactor`: 重構程式碼
- `test`: 測試相關

**範例：**
```bash
git commit -m "feat: 新增學生預約課程功能"
git commit -m "fix: 修正時區顯示錯誤"
git commit -m "docs: 更新 API 文件"
git commit -m "refactor: 簡化預約驗證邏輯"
```

### 🚀 步驟 3：安裝 Node.js & npm

#### 安裝 Node.js

**下載 LTS 版本：**
https://nodejs.org/

**或使用套件管理器：**
```bash
# macOS
brew install node

# Windows: 使用 nvm-windows
# Linux: 使用 nvm
```

**驗證安裝：**
```bash
node --version   # 應該顯示 v18.x.x 或更新版本
npm --version    # 應該顯示 9.x.x 或更新版本
```

#### 初始化 npm 專案

```bash
cd course-booking-system
npm init -y
```

這會產生 `package.json` 檔案：

```json
{
  "name": "course-booking-system",
  "version": "1.0.0",
  "description": "線上教師課程預約系統",
  "main": "index.js",
  "scripts": {
    "test": "jest",
    "test:watch": "jest --watch",
    "test:coverage": "jest --coverage"
  },
  "keywords": ["教育", "預約系統"],
  "author": "你的名字",
  "license": "MIT"
}
```

#### 安裝套件

```bash
# 安裝開發用的測試框架
npm install --save-dev jest

# 安裝程式碼品質工具
npm install --save-dev prettier eslint

# 安裝專案會用到的套件（範例）
npm install express
```

**套件安裝後的資料夾結構：**
```
course-booking-system/
├── node_modules/      # 套件安裝在這裡（不要 commit）
├── package.json       # 專案設定檔
├── package-lock.json  # 鎖定套件版本
└── specs/             # 規格文件
```

**重要：建立 `.gitignore`**
```bash
echo "node_modules/" > .gitignore
echo ".env" >> .gitignore
```

#### npm 常用命令

| 命令 | 說明 | 使用時機 |
|------|------|---------|
| `npm init -y` | 初始化專案 | 建立新專案時 |
| `npm install` | 安裝所有依賴 | clone 專案後 |
| `npm install <package>` | 安裝套件 | 需要新套件時 |
| `npm install -D <package>` | 安裝開發套件 | 安裝測試工具等 |
| `npm test` | 執行測試 | 驗證程式正確性 |
| `npm run <script>` | 執行自訂腳本 | 執行自訂命令 |

### 🧪 步驟 4：設定 Jest 測試框架

#### 安裝 Jest

```bash
npm install --save-dev jest
```

#### 設定 package.json

```json
{
  "scripts": {
    "test": "jest",
    "test:watch": "jest --watch",
    "test:coverage": "jest --coverage"
  }
}
```

#### 建立第一個測試

**建立測試資料夾：**
```bash
mkdir tests
```

**tests/first.test.js:**
```javascript
describe('第一個測試', () => {
  test('測試環境運作正常', () => {
    expect(true).toBe(true);
  });
  
  test('簡單的數學運算', () => {
    expect(2 + 2).toBe(4);
  });
});
```

#### 執行測試

```bash
# 執行所有測試
npm test

# 監視模式（自動重新執行）
npm run test:watch

# 查看測試覆蓋率
npm run test:coverage
```

**成功的輸出：**
```
PASS  tests/first.test.js
  第一個測試
    ✓ 測試環境運作正常 (2 ms)
    ✓ 簡單的數學運算 (1 ms)

Test Suites: 1 passed, 1 total
Tests:       2 passed, 2 total
```

## 🔍 模組 1 總結

### 核心關鍵字

| 術語 | 說明 | 實際用途 |
|------|------|---------|
| **IDE** | 整合開發環境 | VS Code 提供編輯、除錯、擴充功能 |
| **Git** | 版本控制系統 | 追蹤程式碼變更歷史 |
| **Commit** | 提交 | 儲存一個版本快照 |
| **Branch** | 分支 | 獨立開發新功能 |
| **npm** | 套件管理器 | 管理 Node.js 專案依賴 |
| **Jest** | 測試框架 | 自動化測試程式正確性 |

### 工作流程圖

```
1. 撰寫規格 (VS Code)
   ↓
2. 提交版本 (Git)
   ↓
3. 安裝依賴 (npm)
   ↓
4. 撰寫測試 (Jest)
   ↓
5. 推送備份 (GitHub)
```
